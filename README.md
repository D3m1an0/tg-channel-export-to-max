# Перенос канала из Telegram в MAX

Скрипт переносит посты из экспорта Telegram-канала в канал на платформе [MAX](https://max.ru). Поддерживает текстовые сообщения, изображения, видео, аудио и файлы.

[English version](README.en.md)

---

## Требования

- Python 3.8+
- Токен бота MAX
- Экспорт истории Telegram-канала в формате HTML

---

## Шаг 1 — Экспорт истории из Telegram

1. Откройте **Telegram Desktop** (мобильная версия не поддерживает экспорт)
2. Перейдите в нужный канал
3. Нажмите на три точки (меню) в правом верхнем углу → **Экспортировать историю чата**
4. Выберите формат **HTML**
5. Выберите нужный период и типы медиафайлов
6. Нажмите **Экспортировать** — файлы сохранятся в папку (например `ChatExport_2024-01-01`)
7. Внутри папки будут файлы `messages.html`, `messages2.html` и т.д., а также папки с медиафайлами

---

## Шаг 2 — Создание бота MAX и выдача прав

Перенос выполняется через бота платформы MAX. Нужно создать бота и назначить его администратором вашего канала.

1. Откройте приложение **MAX** и найдите бота **@MaxBotAPI** через поиск
2. Напишите ему — бот выдаст вам **токен** (длинная строка вида `xxxxxxxx.xxxxxxxxx`)
3. Сохраните токен — он понадобится на следующем шаге
4. Перейдите в **настройки вашего канала** в MAX → **Администраторы** → добавьте своего бота как администратора с правом **публикации сообщений**
5. Узнайте **ID канала**: отправьте любое сообщение в канал через бота или используйте метод `GET /chats` с вашим токеном — ID будет числовым значением (например `123456789`)

## Шаг 3 — Настройка

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/ВАШ_НИК/tg-to-max.git
   cd tg-to-max
   ```

2. Скопируйте файл с примером переменных окружения:
   ```bash
   cp .env.example .env
   ```

3. Откройте `.env` и заполните значения:
   ```
   MAX_BOT_TOKEN=ваш_токен_бота
   MAX_CHAT_ID=id_вашего_канала
   ```

   - **MAX_BOT_TOKEN** — токен бота, полученный у @MaxBotAPI
   - **MAX_CHAT_ID** — числовой ID канала MAX (бот должен быть администратором канала)

---

## Шаг 4 — Запуск

Тестовый прогон без отправки (рекомендуется сначала):
```bash
python import_tg_to_max.py --base-dir /путь/к/папке/экспорта --dry-run
```

Базовый запуск (читает переменные из `.env`):
```bash
python import_tg_to_max.py --base-dir /путь/к/папке/экспорта
```

Импорт первых 50 сообщений:
```bash
python import_tg_to_max.py --base-dir /путь/к/папке/экспорта --limit 50
```

Импорт с 100-го сообщения:
```bash
python import_tg_to_max.py --base-dir /путь/к/папке/экспорта --offset 100 --limit 50
```

Передать токен напрямую (без `.env`):
```bash
python import_tg_to_max.py --token ваш_токен --chat-id 123456789 --base-dir /путь/к/экспорту
```

---

## Параметры командной строки

| Параметр | По умолчанию | Описание |
|---|---|---|
| `--token` | из `.env` | Токен бота MAX |
| `--chat-id` | из `.env` | ID канала MAX |
| `--base-dir` | `.` | Папка с экспортом Telegram |
| `--files` | `messages.html messages2.html` | HTML-файлы экспорта |
| `--limit` | `10` | Количество сообщений для отправки |
| `--offset` | `0` | Пропустить N первых сообщений |
| `--pause` | `0.6` | Пауза между сообщениями (сек) |
| `--dry-run` | выкл | Тестовый режим без отправки |

---

## Важно

- Бот MAX должен быть **администратором** канала с правом публикации
- Файл `.env` добавлен в `.gitignore` — никогда не коммитьте его
- При большом количестве сообщений используйте `--offset` и `--limit` для пакетного импорта

# Сравнение скриптов миграции

| Критерий | migrate_to_max.py | import_tg_to_max.py |
|----------|-------------------|----------------------|
| **Зависимости** | Нужен `pip install requests beautifulsoup4` | Только стандартная библиотека Python |
| **HTML-парсер** | BeautifulSoup (внешняя библиотека) | Встроенный HTMLParser |
| **Запуск из IDE** | Просто нажать Run — настроек нет | Требует аргументы (`--limit`, `--offset` и др.) |
| **Управление порцией** | Обрабатывает ВСЕ сообщения сразу | Можно задать `--limit` и `--offset` (порции) |
| **Dry-run режим** | Нет | Есть `--dry-run` — проверить без отправки |
| **Типы вложений** | Только фото и видео | Фото, видео, аудио, любые файлы |
| **API endpoint** | `botapi.max.ru` | `platform-api.max.ru` |
| **Возобновление** | `python migrate_to_max.py 50` (с номера) | `--offset 50` |

## Какой использовать из IDE?

Для **обычного запуска из IDE** (PyCharm, VS Code) — **`migrate_to_max.py`**

✅ Просто нажимаешь Run  
✅ Нет обязательных аргументов  
✅ Читает `.env`, берёт все `messages*.html` и отправляет всё  

> ⚠️ Но сначала нужно установить зависимость:
> ```bash
> pip install requests beautifulsoup4
> ```
